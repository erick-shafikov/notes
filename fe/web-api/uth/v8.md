# heap

Память JavaScript — показывает объём Heap по каждой вкладке

Данные и переменные используемые в js хранятся в куче heap, она делится на пространства:
- read-only-space - данные которые невозможно изменить типа false true
- new-space - v8 делит данные по поколения - старое и молодое. движок делает предположение, что те данные которые существую давно с меньшей долей вероятности будут удалены
- old-space - если данные молодого поколения переживают несколько сборок мусора, то отправляются сюда
- Code space - хранение исполняемого кода
- Large object space - данные превышающие лимит kMaxRegularHeapObjectSize
- New large object space - данные молодого поколения
- Old large object space
- Code large object space 

Максимальный объем кучи варьируется по устройствам

# gc

Сборка мусора, опирается на механизмы:
- Minor Garbage Collector или Scavenger - реализует алгоритм Cheney, работает с полупространства From space и To space


Корни GC - глобальная область видимости JS, DOM-дерево или контекст отладчика. Проход переносит объекты из from space в to space, удаляются все кто остался в from space. Заполняется to space пока не переполнится
