# Стохастический градиентный спуск (Stochastic gradient descent SGD)

Метод Робинса-Монро. Так как признаков может быть много и матрица $(X^T X)$ может быть вырожденной или плохо обусловленной, а также её обращение вычислительно затратно, то вместо аналитического решения $w = (X^T X)^{-1} X^T y$ используют градиентные алгоритмы. Полный градиентный шаг $w^{t+1} = w^{t} - η * \sum_{i=1}^{l}∇L_i(w^{t}, x_i)$ требует прохода по всей выборке и является дорогим, поэтому его заменяют стохастической оценкой градиента. Как вариант, брать по одному случайному объекту выборки $w^{t+1} = w^{t} - η ∇L_k(w^{t}, x_k)$. При этом математическое ожидание такого градиента совпадает с полным градиентом.

Алгоритм:

- Вход:
- - X - выборка, η - шаг обучения, λ - коэффициент сглаживания
- Выход:
- - ω - вектор весовых коэффициентов
- Инициализация весов ω некоторыми начальными значениями
- Вычисление функционала качества $Q(ω) = 1 / l * \sum_{i=1}^{l}L_i(ω)$
- Цикл
- - случайный выбор $x_k$ из $X^l$
- - Вычисление функции потерь $ε_k = L_k(ω)$
- - Шаг стохастического градиентного алгоритма $ω = ω - η * ∇L_k(ω, x_k)$
- - [*] Пересчет функционала качества $Q = λ * ε_k + (1 - λ) * Q$
- - Пока Q и/или не достигнут заданных значений

Используется скользящая средняя (экспоненциальная средняя). для обычной средней $Q_m = (1 / m) * ε_m + (1 / m) * ε_{m - 1} + (1 / m) * ε_{m - 2} + ... + (1 / m) * ε_1$ если домножить на m и перенести $ε_m$ $m * Q_m - ε_m = ε_{m - 1} + ε_{m - 2} + ... + ε_1$, тогда $Q_{m - 1} = 1 / (m - 1) * (m * Q_m - ε_m) = m / (m - 1) * Q_m - 1 / (m - 1) * ε_m$ то есть пересчитываем среднее с появлением нового слагаемого. Тогда $Q_m = λ ε_m + (1 - λ) Q_{m-1}$ вес имеет последнее, первые элементы затухают. Пример $λ = 2 / (N + 1)$ если $N = 99$, то $λ = 0.02$ то есть учитываем последние 99 элементов, усредняем по последним $1/λ$

## шаг обучения

Для обучения можно взять не константу а переменную η $η_t = η_0(1 - t / T)$ или экспоненциальную $η = η_0 * e ^ {- 1 / T}$. Используется Learning rate scheduling - это стратегия изменения шага обучения во времени для более быстрой и стабильной сходимости.
Общее обновление с переменным learning rate: $w_{t+1} = w_t - η * ∇L(w_t)$

Step decay: $η = η_0 * γ^{t / k}$

Exponential decay: $η = η_0 * e^{-kt}$

Cosine annealing: $η = η_{min} + 1/ 2(η_{max} - η_{min}) (1 + cos(t/T\π))$

Linear decay: $η = η_0 (1 - t /T)$

Warmup (линейный рост): $η = η_0 * t / T_{warmup}$

Reduce on plateau (идея, не строгая формула):

$η_{t+1} =
\begin{cases}
η * γ, & \text{если метрика не улучшается} \\
η, & \text{иначе}
\end{cases}$

## инициализация весов

- нули — допустимо для линейных моделей, но плохо для нейросетей (симметрия нейронов)
- малые случайные значения $ω_i = rand(-ε, ε)$, где ε — небольшое число
- использование специальных схем инициализации (в зависимости от модели)
- предварительное обучение на малой выборке (warm start)
- мультистарт — несколько случайных инициализаций с выбором лучшего решения
  объекты выборки

## Рекомендации объектам выборки:

- случайная перетасовка (shuffle) перед каждой эпохой
- равновероятный выбор объектов (классический SGD)
- возможно увеличение вероятности выбора объектов с большой ошибкой (importance sampling)
- в задачах классификации — можно чаще выбирать объекты с малым margin
- избегать полного исключения “хороших” или “плохих” объектов (важен баланс)

## Выбор шага

- убывающий шаг $h_t → 0$, например $h_t = 1 / t$
- условия сходимости (Роббинс-Монро): $\sum h_t → ∞$, $\sum h_t^2 < ∞$
- шаг может быть постоянным (на практике часто используется)
- возможны адаптивные методы выбора шага (RMSProp, Adam и др.)
- для квадратичных функций возможен выбор шага $h ≈ 1 / ||x_i||^2$ (частный случай)

Плюсы SGD:

- малый объем вычислений
- легкая реализация для разных функций потерь
- возможна реализация на непрерывном потоке данных
- можно использовать только часть данных для обучения
- использование в задачах big data

Минусы SGD:

- неплавный спуск по сравнению с градиентным спуском
- высокая вариативность (шум) обновлений параметров
- требуется аккуратный подбор шага обучения η (например, убывающего по условиям Робинса-Монро)
- застревание в локальных минимумах
- побор гиперпараметров (шаг обучения, скорость забывания, критерий остановы)
- переобучаемость модели

```python
import numpy as np


def func(x):
    return 0.5 * x ** 2 - 0.1 / np.exp(-x) + 0.5 * np.cos(2 * x) - 2.


def model(x, w):
    return w[0] + w[1] * x + w[2] * x ** 2 + w[3] * np.cos(2 * x) + w[4] * np.sin(2 * x)


def loss(y_true, y_pred):
    return (y_true - y_pred) ** 2


def gradients(x, y_true, w):
    y_pred = model(x, w)
    error = y_pred - y_true
    return 2 * error * np.array([1, x, x ** 2, np.cos(2 * x), np.sin(2 * x)])


coord_x = np.arange(-5.0, 5.0, 0.1)  # Значения по оси абсцисс [-5; 5] с шагом 0.1
coord_y = func(coord_x)  # Значения функции по оси ординат

sz = len(coord_x)  # Количество значений функций (точек)
eta = np.array([0.01, 0.001, 0.0001, 0.01, 0.01])  # Разные шаги обучения для каждого параметра w0, w1, w2, w3, w4
w = np.array([0., 0., 0., 0., 0.])  # Начальные значения параметров модели
N = 500  # Число итераций градиентного алгоритма SGD
lm = 0.02  # Значение параметра лямбда для вычисления скользящего экспоненциального среднего
Qe = np.mean(loss(coord_y, model(coord_x, w)))  # начальное значение скользящего среднего эмпирического риска
np.random.seed(0)  # генерация одинаковых последовательностей псевдослучайных чисел

# Стохастический градиентный спуск
for i in range(N):
    k = np.random.randint(sz - 1)  # выбор случайного индекса
    x_k = coord_x[k]
    y_k = coord_y[k]

    # Вычисление градиентов
    grads = gradients(x_k, y_k, w)

    # Обновление весов с учетом разных шагов обучения для каждого параметра
    w -= eta * grads

    # Обновление скользящего среднего эмпирического риска
    Qe = lm * loss(y_k, model(x_k, w)) + (1 - lm) * Qe

# Финальная ошибка после всех итераций по всей выборке
Q = np.mean(loss(coord_y, model(coord_x, w)))

```

# Метод Ньютона-Рафсона

Метод используется для поиска корня уравнения $f(x) = 0$ или минимума функции. Идея в том, что функция в текущей точке аппроксимируется касательной (линейным приближением), и следующая точка берется как пересечение этой касательной с осью X.

Формула итерации для поиска корня: $x_{n+1} = x_n - f(x_n) / f'(x_n)$. Метод использует значение функции и ее производную в текущей точке.

Для задачи оптимизации (поиск минимума) используется производная функции: $x_{n+1} = x_n - f'(x_n) / f''(x_n)$. Здесь используется первая и вторая производные функции.

В многомерном случае используется градиент и матрица вторых производных (гессиан): $x_{n+1} = x_n - H^{-1} * grad f(x_n)$, где $H$ — матрица вторых производных.

Сходимость метода быстрее, чем у градиентного спуска (квадратичная), но вычислительно дороже, так как требуется считать вторые производные. Требует хорошего начального приближения и гладкости функции.
