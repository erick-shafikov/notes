PCA эффективен прежде всего тогда, когда в признаковом пространстве есть **линейные зависимости** — когда один признак приблизительно выражается через остальные:

$$f_j(x) = \sum_{\substack{i=1 \\ i \neq j}}^n a_i\, f_i(x) + a_0$$

Если таких зависимостей нет, собственные числа матрицы Грама все положительны и далеки от нуля — PCA не даёт выигрыша. Если зависимость есть, одно из собственных чисел окажется равным нулю или близким к нему.

**Пример.** Пусть $x_i, y_i \sim \mathcal{N}(0, 1)$ и определены три признака: $f_1(x_i) = x_i$, $f_2(y_i) = y_i$, $f_3(x_i, y_i) = 0.5(x_i + y_i)$. Третий признак — линейная комбинация первых двух: $f_3 = 0.5 f_1 + 0.5 f_2$. Матрица признаков:

$$F = \begin{bmatrix} f_1(x_1) & f_2(y_1) & f_3(x_1, y_1) \\ \vdots & \vdots & \vdots \\ f_1(x_l) & f_2(y_l) & f_3(x_l, y_l) \end{bmatrix}_{l \times 3}$$

Матрица Грама $\hat{F} = \frac{1}{l} F^T F$ будет вырожденной (или почти вырожденной), и характеристическое уравнение $\det(\hat{F} - \lambda I) = 0$ даст одно собственное число равное или близкое к нулю. Соответствующее направление $u_j$ не несёт информации — его можно отбросить, задав порог $\theta$: компоненты с $\lambda_j \leq \theta$ исключаются. Матрица новых признаков:

$$G = F \cdot U^T, \qquad U = [u_1 \mid u_2 \mid \cdots \mid u_n]^T$$

После отбора $m$ главных компонент размерность снижается с $n$ до $m$.

Таким образом, с переобучением модели малой выборкой можно бороться двумя способами:

- L-регуляризация (L1 или L2)
- метод главных компонент

---

**Связь PCA и L2-регуляризации через мультиколлинеарность.** Рассмотрим задачу регрессии с квадратичными потерями:

$$Q(a, X^l) = \frac{1}{2}\sum_{i=1}^l \bigl(y_i - \langle\omega, x_i\rangle\bigr)^2, \qquad a(x_i) = \langle\omega, x_i\rangle$$

В матричном виде, где $y = [y_1, \ldots, y_l]^T$ — вектор целевых значений, $\omega = [\omega_1, \ldots, \omega_n]^T$ — вектор весов, $X$ — матрица объектов размера $l \times n$:

$$Q(a, X^l) = \frac{1}{2}(y - X\omega)^T(y - X\omega)$$

Приравнивая градиент по $\omega$ к нулю, получаем нормальное уравнение:

$$\frac{\partial Q}{\partial \omega} = 0 \quad \Longrightarrow \quad \omega^* = (X^T X)^{-1} X^T y$$

Проблема возникает при **мультиколлинеарности**: если признаки линейно зависимы или близки к тому, матрица $X^T X$ становится плохо обусловленной (почти вырожденной). Обращение такой матрицы даёт решение с огромными по модулю коэффициентами $\omega^*$ — модель плохо обобщается.

**L2-регуляризация (ridge)** решает это, добавляя $\tau I$ к матрице — диагональные элементы «приподнимаются» на $\tau > 0$, что гарантирует обратимость:

$$\omega^* = (X^T X + \tau I)^{-1} X^T y$$

**PCA** решает ту же проблему иначе: проецирует данные на $m$ главных компонент, убирая направления с нулевой или малой дисперсией — именно они порождают мультиколлинеарность. Рабочий процесс: вычислить $\hat{F} = \frac{1}{l} F^T F$, найти $U$ из собственных векторов, перейти в новый базис $G = X \cdot U^T$, затем обучить линейную модель на $G$:

$$\omega^* = (G^T G)^{-1} G^T y, \qquad \text{predict} = G\,\omega^*$$

И L2-регуляризация, и PCA избавляют от мультиколлинеарности: первая — сглаживая спектр $X^T X$, вторая — явно отбрасывая «нулевые» направления.

```python
import numpy as np

np.random.seed(0)

n_total = 1000  # число образов выборки
n_features = 200  # число признаков

table = np.zeros(shape=(n_total, n_features))

for _ in range(100):
    i, j = np.random.randint(0, n_total), np.random.randint(0, n_features)
    table[i, j] = np.random.randint(1, 10)

# здесь продолжайте программу
N = len(table)
F = 1 / N * table.T @ table
L, W = np.linalg.eig(F)

WW = sorted(zip(L, W), key=lambda lx: lx[0], reverse=False)
WW = np.array([w[1] for w in WW])

# в новом базисе
data_x = table @ WW.T
# фильтрация по массиву L
data_x = data_x[:, L >= 0.01]
```

```python
import numpy as np


def func(x):
    return 0.5 * x + 0.2 * x ** 2 - 0.05 * x ** 3 + 0.2 * np.sin(4 * x) - 3


coord_x = np.arange(-4.0, 6.0, 0.1)
coord_y = func(coord_x)
K = 10
X = np.array([[xx ** i for i in range(K)] for xx in coord_x])  # обучающая выборка для поиска коэффициентов модели
Y = coord_y

X_train = X[::2]  # обучающая выборка (входы)
Y_train = Y[::2]  # обучающая выборка (целевые значения)

# здесь продолжайте программу
N = len(X_train)
F = 1 / N * X_train.T @ X_train
L, W = np.linalg.eig(F)

WW = sorted(zip(L, W), key=lambda lx: lx[0], reverse=False)
WW = np.array([w[1] for w in WW])

# вычислите новые признаки G матрицы X в пространстве собственных векторов матрицы WW
G = X @ WW.T
# Оставьте в матрице G только первые 7 признаков
G = G[:, :7]
# Сформируйте матрицу XX_train из образов с новыми признаками G командой:
XX_train = G[::2]
# На основе выборки XX_train, Y_train вычислите вектор параметров w
w = np.linalg.inv(XX_train.T @ XX_train) @ XX_train.T @ Y_train
# Выполните восстановление функции func используя матрицу G и вектор параметров w
predict = G @ w
```
