# Метрики ранжирования

Задача ранжирования отличается от классификации: нужно не просто разделить объекты на классы, а упорядочить их так, чтобы релевантные оказались выше нерелевантных. Типичные примеры — поисковые системы, рекомендации, информационный поиск. Метрики здесь оценивают качество **порядка** выдачи, а не точности отдельных предсказаний.

Формально: для каждого запроса $q$ модель выдаёт список из $n$ объектов, упорядоченных по убыванию скора. Каждый объект $d_i$ имеет метку релевантности $y_i \geq 0$. При бинарной релевантности $y_i \in \{0, 1\}$, при градуированной — $y_i \in \{0, 1, 2, \ldots\}$.

## Precision@k

Простейшая метрика: доля релевантных объектов среди первых $k$ позиций выдачи:

$$\text{P@}k = \frac{1}{k} \sum_{i=1}^{k} \mathbb{1}[y_i = 1]$$

Не учитывает порядок внутри топ-$k$: объект на первой позиции и объект на $k$-й вносят одинаковый вклад. Работает только с бинарной релевантностью.

## Mean Average Precision (MAP)

Average Precision (AP) для одного запроса — это среднее значение P@k, вычисленное только в позициях, где стоит релевантный объект:

$$\text{AP} = \frac{1}{R} \sum_{k=1}^{n} \text{P@}k \cdot \mathbb{1}[y_k = 1]$$

где $R = \sum_i \mathbb{1}[y_i = 1]$ — общее число релевантных объектов. AP растёт, когда релевантные объекты стоят ближе к началу списка.

**MAP** — среднее AP по всем запросам:

$$\text{MAP} = \frac{1}{|Q|} \sum_{q \in Q} \text{AP}(q)$$

Чувствителен к позициям всех релевантных объектов, но не учитывает степень релевантности — только бинарную метку.

## Normalized Discounted Cumulative Gain (NDCG)

NDCG работает с **градуированной релевантностью** и явно учитывает позицию: объект на первой позиции ценнее, чем на пятой.

**Discounted Cumulative Gain** — сумма релевантностей с логарифмическим штрафом за позицию:

$$\text{DCG@}k = \sum_{i=1}^{k} \frac{2^{y_i} - 1}{\log_2(i + 1)}$$

где $y_i$ — релевантность объекта на позиции $i$, знаменатель $\log_2(i+1)$ — дисконт: первая позиция имеет дисконт $\log_2 2 = 1$ (без штрафа), вторая — $\log_2 3 \approx 1.58$, и так далее.

Для нормировки вводится идеальный DCG (IDCG) — DCG при оптимальном упорядочивании (по убыванию релевантности). Тогда:

$$\text{NDCG@}k = \frac{\text{DCG@}k}{\text{IDCG@}k} \in [0, 1]$$

NDCG = 1 означает, что модель выдала объекты в идеальном порядке по релевантности.

Пример расчёта для $k=4$ при релевантностях $[3, 2, 3, 0]$ (предсказанный порядок) и $[3, 3, 2, 0]$ (идеальный):

$$\text{DCG} = \frac{2^3-1}{\log_2 2} + \frac{2^2-1}{\log_2 3} + \frac{2^3-1}{\log_2 4} + \frac{2^0-1}{\log_2 5} = 7 + \frac{3}{1.58} + \frac{7}{2} + 0 \approx 12.40$$

$$\text{IDCG} = \frac{7}{1} + \frac{7}{1.58} + \frac{3}{2} + 0 \approx 13.93, \qquad \text{NDCG} \approx \frac{12.40}{13.93} \approx 0.89$$

## Mean Reciprocal Rank (MRR)

Метрика для задач, где нужен **один** правильный ответ (вопросно-ответные системы, поиск дубликатов). Reciprocal Rank — обратная позиция первого релевантного объекта:

$$\text{RR} = \frac{1}{\text{rank}_1}$$

где $\text{rank}_1$ — позиция первого релевантного объекта в списке. **MRR** — среднее по запросам:

$$\text{MRR} = \frac{1}{|Q|} \sum_{q \in Q} \frac{1}{\text{rank}_1(q)}$$

Если релевантный объект стоит первым — RR = 1; вторым — RR = 0.5; десятым — RR = 0.1. Метрика полностью игнорирует второй и последующие релевантные объекты.

## Сравнение метрик

| Метрика | Градуированная релевантность | Учитывает позицию | Несколько релевантных |
| ------- | ---------------------------- | ----------------- | --------------------- |
| P@k     | ✗                            | ✗                 | ✓                     |
| MAP     | ✗                            | ✓                 | ✓                     |
| NDCG    | ✓                            | ✓                 | ✓                     |
| MRR     | ✗                            | ✓                 | ✗                     |

На практике: NDCG — стандарт для поисковых систем и рекомендаций с оценками релевантности; MAP — для информационного поиска с бинарными метками; MRR — для задач поиска одного правильного ответа.
