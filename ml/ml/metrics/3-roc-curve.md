Precision и recall зависят от конкретного выбранного порога: сменил порог — получил другие числа. ROC-кривая (Receiver Operating Characteristic) решает эту проблему: она показывает поведение классификатора при **всех** возможных порогах одновременно и отвечает на вопрос о качестве **ранжирования** — насколько хорошо модель вообще разделяет классы, независимо от того, где провести границу.

Модель выдаёт скор $b(x) = \langle \omega, x \rangle$ — чем он выше, тем увереннее модель в принадлежности к положительному классу. Порог $t$ переводит скор в метку: $a(x, t) = \text{sign}(\langle \omega, x \rangle - t)$. Снижая $t$, мы относим всё больше объектов к положительному классу — находим больше реальных положительных (TPR растёт), но и захватываем больше отрицательных (FPR растёт). Повышая $t$ — делаем обратное. Идеальная модель — та, у которой скоры положительных объектов всегда выше, чем у отрицательных: $b(x_i) > b(x_j)$ при $y_i = +1,\ y_j = -1$. Тогда существует порог, при котором FPR = 0 и TPR = 1 одновременно.

При фиксированном пороге $t$ классификатор характеризуется двумя числами.

**FPR** (False Positive Rate) — какую долю истинно отрицательных объектов модель ошибочно записала в положительные. Знаменатель — только отрицательные объекты, поэтому FPR не зависит от числа положительных:

$$\text{FPR}(t) = \frac{FP(t)}{FP(t) + TN(t)} = \frac{\sum_{i=1}^l [y_i = -1]\,[a(x_i, t) = +1]}{\sum_{i=1}^l [y_i = -1]}$$

**TPR** (True Positive Rate, он же recall) — какую долю истинно положительных объектов модель нашла. Знаменатель — только положительные объекты:

$$\text{TPR}(t) = \frac{TP(t)}{TP(t) + FN(t)} = \frac{\sum_{i=1}^l [y_i = +1]\,[a(x_i, t) = +1]}{\sum_{i=1}^l [y_i = +1]}$$

$1 - \text{FPR}(t)$ называется специфичностью: доля отрицательных, верно отклонённых моделью. Хороший классификатор имеет высокий TPR при низком FPR — точка на ROC-плоскости находится в левом верхнем углу.

![Распределения скоров и точки на ROC при разных порогах](images/roc-threshold.png)

_Левый график: синие — распределение скоров положительного класса, красные — отрицательного. Вертикальные линии — три значения порога $t$. Чем правее порог, тем меньше объектов попадает в «положительные» — FPR и TPR оба падают. Правый график: каждому порогу соответствует одна точка $(FPR, TPR)$ на ROC-плоскости. Сдвиг порога влево (снижение $t$) движет точку вправо вверх — захватываем больше объектов обоих классов._

**Построение ROC-кривой** сводится к одному проходу по отсортированным объектам без перебора порогов в явном виде.

Шаг 1. Вычислить скор $b(x_i) = \langle \omega, x_i \rangle$ для каждого объекта и отсортировать по убыванию: $b(x_1) \geq b(x_2) \geq \ldots \geq b(x_l)$. Это эквивалентно последовательному снижению порога — на каждом шаге порог опускается ровно до следующего объекта.

Шаг 2. Начать в точке $(0, 0)$ — порог выше всех скоров, никто не классифицирован как положительный.

Шаг 3. Для каждого объекта в порядке убывания скора: если $y_i = +1$ — сделать шаг **вверх** (TPR увеличивается на $1 / l_+$, где $l_+$ — число положительных объектов); если $y_i = -1$ — сделать шаг **вправо** (FPR увеличивается на $1 / l_-$, где $l_-$ — число отрицательных). Интуиция: встретили положительный объект на высоком скоре — хорошо, TPR растёт без штрафа по FPR; встретили отрицательный — плохо, FPR растёт впустую.

Шаг 4. После всех объектов кривая заканчивается в точке $(1, 1)$.

![Таблица ранжирования и построение шаговой кривой](images/roc-ranking.png)

_Левая таблица: объекты отсортированы по убыванию $b(x_i)$; синие строки ($y_i=+1$) поднимают TPR, красные ($y_i=-1$) сдвигают FPR вправо. Правый график: итоговая ступенчатая кривая — длинные вертикальные участки означают, что модель нашла много положительных объектов подряд без ошибок._

![ROC-кривые: идеальная, реальная и случайная](images/roc-curves.png)

_Идеальная модель (красная): все положительные объекты стоят первыми в ранжировании — кривая сразу идёт вертикально вверх до $(0, 1)$, затем горизонтально до $(1, 1)$. Случайная модель (пунктир): положительные и отрицательные перемешаны случайно — при любом пороге FPR ≈ TPR, кривая ложится на диагональ. Реальная модель (синяя): между ними; чем дальше кривая от диагонали в сторону левого верхнего угла, тем лучше модель разделяет классы._

**AUC-ROC** (Area Under Curve) — площадь под ROC-кривой, $0 \leq \text{AUC} \leq 1$. Это агрегированная мера качества при всех порогах сразу: не нужно выбирать конкретный порог, чтобы сравнить две модели. Вероятностная интерпретация: AUC — это вероятность того, что случайно выбранный положительный объект получит более высокий скор $b(x)$, чем случайно выбранный отрицательный. AUC = 1 означает, что модель идеально разделяет классы; AUC = 0.5 — модель не лучше случайного угадывания; AUC < 0.5 — модель систематически ошибается (что само по себе полезно: инвертируй предсказания).

**Коэффициент Джини** — нормированная версия AUC, которая переводит шкалу из $[0.5, 1]$ в $[0, 1]$:

$$G = 2 \cdot \text{AUC-ROC} - 1, \qquad G \in [-1, 1]$$

Для идеальной модели $G = 1$, для случайной $G = 0$. Метрика чувствительна к балансу классов и плохо обобщается на многоклассовые выборки.

- вычисления FPR TPR

```python
import numpy as np
from sklearn import svm
from sklearn.model_selection import train_test_split

np.random.seed(0)

# исходные параметры распределений классов
r1 = 0.2
D1 = 3.0
mean1 = [2, -2]
V1 = [[D1, D1 * r1], [D1 * r1, D1]]

r2 = 0.5
D2 = 2.0
mean2 = [-1, -1]
V2 = [[D2, D2 * r2], [D2 * r2, D2]]

# моделирование обучающей выборки
N1 = 1000
N2 = 1000
x1 = np.random.multivariate_normal(mean1, V1, N1).T
x2 = np.random.multivariate_normal(mean2, V2, N2).T

data_x = np.hstack([x1, x2]).T
data_y = np.hstack([np.ones(N1) * -1, np.ones(N2)])

x_train, x_test, y_train, y_test = train_test_split(data_x, data_y, random_state=123, test_size=0.5, shuffle=True)

clf = svm.SVC(kernel='linear')
clf.fit(x_train, y_train)
predict = clf.predict(x_test)

w = [clf.intercept_[0], *clf.coef_[0]]

t = 2

y_pred = np.sign(w[1:] @ x_test.T + w[0] - t)

TP = np.sum((y_pred == 1) & (y_test == 1))
TN = np.sum((y_pred == -1) & (y_test == -1))
FP = np.sum((y_pred == 1) & (y_test == -1))
FN = np.sum((y_pred == -1) & (y_test == 1))

FPR = FP / (FP + TN)
TPR = TP / (TP + FN)
```

```python
import numpy as np
from sklearn import svm
from sklearn.model_selection import train_test_split

np.random.seed(0)

# исходные параметры распределений классов
r1 = -0.2
D1 = 3.0
mean1 = [1, -5]
V1 = [[D1, D1 * r1], [D1 * r1, D1]]

r2 = 0.5
D2 = 2.0
mean2 = [-1, -2]
V2 = [[D2, D2 * r2], [D2 * r2, D2]]

# моделирование обучающей выборки
N1 = 1000
N2 = 1000
x1 = np.random.multivariate_normal(mean1, V1, N1).T
x2 = np.random.multivariate_normal(mean2, V2, N2).T

data_x = np.hstack([x1, x2]).T
data_y = np.hstack([np.ones(N1) * -1, np.ones(N2)])

x_train, x_test, y_train, y_test = train_test_split(data_x, data_y, random_state=123, test_size=0.5, shuffle=True)

clf = svm.SVC(kernel='linear')
clf.fit(x_train, y_train)

w = [clf.intercept_[0], *clf.coef_[0]]

FPR = []
TPR = []

for t in np.arange(5.7, -7.8, -0.1):
    y_pred = np.sign(w[1:] @ x_test.T + w[0] - t)

    TP = np.sum((y_pred == 1) & (y_test == 1))
    TN = np.sum((y_pred == -1) & (y_test == -1))
    FP = np.sum((y_pred == 1) & (y_test == -1))
    FN = np.sum((y_pred == -1) & (y_test == 1))

    FPR += [FP / (FP + TN)]
    TPR += [TP / (TP + FN)]
```
