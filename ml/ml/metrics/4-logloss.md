# Log Loss

Accuracy, Precision, Recall и F1 оценивают метки — жёсткие решения модели. Но часто модель выдаёт **вероятности**: $\hat{p}_i = P(y=1 \mid x_i)$, и качество этих вероятностей стоит оценивать отдельно. Log Loss (логарифмическая потеря, binary cross-entropy) штрафует не за то, правильна ли метка, а за то, насколько уверенно модель давала правильный ответ.

## Определение

Для бинарной классификации $y_i \in \{0, 1\}$:

$$\text{LogLoss} = -\frac{1}{l} \sum_{i=1}^{l} \bigl[y_i \log \hat{p}_i + (1 - y_i) \log(1 - \hat{p}_i)\bigr]$$

где $\hat{p}_i \in (0, 1)$ — предсказанная вероятность класса $1$ для объекта $x_i$, $l$ — размер выборки.

Каждое слагаемое выбирает нужную ветку: при $y_i = 1$ штраф равен $-\log \hat{p}_i$ — чем выше уверенность в правильном ответе, тем меньше потеря; при $y_i = 0$ штраф равен $-\log(1 - \hat{p}_i)$.

## Интуиция

Логарифм превращает уверенность в штраф нелинейно. Если модель говорит $\hat{p} = 0.9$ на правильном объекте — штраф $-\log 0.9 \approx 0.1$ небольшой. Если $\hat{p} = 0.1$ на правильном объекте — штраф $-\log 0.1 \approx 2.3$ большой. Если $\hat{p} \to 0$ на правильном объекте — штраф $\to \infty$: излишне самоуверенная ошибка карается очень жёстко.

| $\hat{p}$ (при $y=1$) | Штраф $-\log \hat{p}$ |
| --------------------- | --------------------- |
| 0.99                  | 0.01                  |
| 0.9                   | 0.11                  |
| 0.5                   | 0.69                  |
| 0.1                   | 2.30                  |
| 0.01                  | 4.61                  |

Идеальная модель имеет LogLoss = 0. Модель, предсказывающая $\hat{p} = 0.5$ для всех объектов (полная неопределённость), получает LogLoss = $\log 2 \approx 0.693$ — это базовый уровень для сравнения.

## Связь с MLE и кросс-энтропией

Log Loss — это в точности отрицательный log-likelihood Bernoulli-распределения, делённый на $l$:

$$\text{LogLoss} = -\frac{1}{l} \log L(w) = -\frac{1}{l} \sum_{i=1}^l \log P(y_i \mid x_i, w)$$

Минимизация LogLoss при обучении и оценка Log Loss на тесте — это один и тот же функционал, что делает метрику особенно естественной для логистической регрессии и других вероятностных моделей.

## Многоклассовый случай

Обобщение на $K$ классов — categorical cross-entropy:

$$\text{LogLoss} = -\frac{1}{l} \sum_{i=1}^{l} \sum_{k=1}^{K} \mathbb{1}[y_i = k] \cdot \log \hat{p}_{ik}$$

где $\hat{p}_{ik}$ — предсказанная вероятность класса $k$ для объекта $i$. Каждый объект вносит только один ненулевой член — логарифм вероятности истинного класса.

## Когда использовать

Log Loss подходит, когда нужны откалиброванные вероятности, а не только правильные метки: в задачах ранжирования, медицинской диагностике, кредитном скоринге — везде, где важна степень уверенности модели, а не только бинарное решение. В отличие от Accuracy и F1, Log Loss наказывает за уверенные ошибки, что стимулирует модель избегать экстремальных предсказаний без оснований.
