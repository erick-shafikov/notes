# Bootstrap, Bagging и Случайный лес

Бинарные решающие деревья в чистом виде не применяются из-за высокой дисперсии и переобучения. На практике их используют в **ансамблевых методах**: bagging и boosting. Сложность по памяти ансамбля — $O(Nk)$, где $k$ — число деревьев.

## Базовый алгоритм в ансамблях

В ансамблевых методах термин **базовый алгоритм** $b_t(x)$ обозначает отдельную модель, из которых складывается итоговый ансамбль. Это не один пороговый сплит $[f_j(x) \leq q]$, а полноценная модель — пороговые предикаты являются лишь внутренними узлами внутри неё.

Иерархия для дерева:

```
Ансамбль  a(x) = Σ αt · bt(x)
                       ↑
           базовый алгоритм bt — решающее дерево
                                        ↑
                               узел — предикат [fj(x) ≤ q]
```

В роли базового алгоритма может выступать любая обучаемая модель:

* **Решающий пень** (stump, глубина 1) — одно пороговое разделение; применяется в AdaBoost для ускорения и интерпретируемости
* **Решающее дерево** глубины 2–8 — наиболее распространённый выбор в gradient boosting (XGBoost, LightGBM)
* **Линейная модель** — регрессия или классификатор; ансамбль линейных базовых алгоритмов строит кусочно-линейную аппроксимацию
* **Нейронная сеть** — теоретически допустима, но редко применяется из-за высокой стоимости обучения на каждом шаге
* **Метод ближайших соседей (kNN)** — теоретически допустим, но не дифференцируем и плохо масштабируется

На практике в бустинге почти всегда используют **неглубокие решающие деревья**: они достаточно гибки, чтобы уловить нелинейности, но слабы настолько, чтобы ансамблю было что улучшать.

## Bagging (Bootstrap Aggregating)

Bagging строит $T$ независимых моделей $a_1(x), \ldots, a_T(x)$ на разных подвыборках и усредняет их прогнозы:

$$a(x) = \frac{1}{T}\sum_{t=1}^{T} a_t(x)$$

**Почему усреднение снижает ошибку.** Пусть каждая модель даёт прогноз $x_t = \theta + \eta_t$, где $\theta$ — истинное значение, $\eta_t$ — независимый шум с дисперсией $\sigma^2$ и $\mathbb{E}[\eta_t] = 0$, $\mathbb{E}[\eta_i \eta_j] = 0$ при $i \neq j$. Ошибка усреднённого прогноза:

$$\mathbb{E}\!\left[\left(\theta - \frac{1}{T}\sum_{t=1}^{T} x_t\right)^{\!2}\right] = \frac{\sigma^2}{T}$$

то есть при независимых моделях дисперсия ошибки снижается в $T$ раз.

**Формирование подвыборок — bootstrap.** Из исходной выборки $X^l = \{(x_i, y_i)\}_{i=1}^l$ формируется $T$ подвыборок длиной $m$ с возвращением (sampling with replacement): каждый объект может попасть в одну выборку несколько раз или не попасть вовсе. Объекты, не вошедшие ни в одну выборку $U_t$, образуют **out-of-bag** множество.

## Критерий out-of-bag

Оценка качества без отдельной тестовой выборки — усреднение прогнозов только по тем деревьям, в обучение которых объект $x_i$ не входил:

$$\sum_{i=1}^{l} \left[\,\mathrm{sign}\!\left(\sum_{t=1}^{T} [x_i \notin U_t]\, b_t(x_i)\right) \neq y_i\,\right] \to \min$$

где $[x_i \notin U_t]$ — индикатор того, что $x_i$ не вошёл в $t$-ю bootstrap-выборку.

## Случайный лес

Bagging на деревьях устраняет дисперсию, но деревья остаются коррелированными: при наличии сильного признака все деревья будут делать разбиение по нему. **Случайный лес** добавляет дополнительную рандомизацию: на каждом шаге разбиения узла перебираются не все $n$ признаков, а случайно выбранное подмножество размера $m$:

$$m = \lfloor\sqrt{n}\rfloor \quad \text{(классификация)}, \qquad m = \lfloor n/3 \rfloor \quad \text{(регрессия)}$$

Итоговый прогноз:

$$a(x) = \frac{1}{T}\sum_{t=1}^{T} a_t(x) \quad \text{(регрессия)}, \qquad a(x) = \mathrm{sign}\!\left(\frac{1}{T}\sum_{t=1}^{T} a_t(x)\right) \quad \text{(классификация)}$$

При увеличении числа деревьев $T$ дисперсия ошибки монотонно убывает и стабилизируется: расхождение при достаточно большом $T$ остаётся константным.

Преимущества:

- высокая точность прогнозов
- не чувствителен к выбросам и масштабированию признаков
- не требует тонкой настройки гиперпараметров
- способен работать с большим числом признаков
- редко переобучается
- работает с пропущенными данными
- легко масштабируется и распараллеливается

Недостатки:

- сложно интерпретировать (нет одного дерева с понятными правилами)
- плохо работает при разреженных данных
- не может экстраполировать за пределы обучающей выборки
- возможно переобучение на шумных данных
- склонен смещаться в сторону более частых классов при дисбалансе

- задача по восстановлению с помощью bootstrap метода RandomForestRegressor

```python
import numpy as np
from sklearn.ensemble import RandomForestRegressor

x = np.arange(-3, 3, 0.1)
y = 0.3 * x + np.cos(2 * x) + 0.2 * np.sin(7 * x)  # + np.random.normal(0.0, 0.1, n_samples)
x = x.reshape(-1, 1)

T = 5  # число деревьев

rf = RandomForestRegressor(max_depth=8, n_estimators=T, random_state=1)
clf = rf.fit(x, y)
pr_y = rf.predict(x)

Q = np.square(pr_y - y).mean()
```

- задачи бинарной классификации

```python
import numpy as np
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import train_test_split

np.random.seed(0)
n_feature = 5

# исходные параметры для формирования образов обучающей выборки
r1 = 0.7
D1 = 3.0
mean1 = [3, 7, -2, 4, 6]
V1 = [[D1 * r1 ** abs(i - j) for j in range(n_feature)] for i in range(n_feature)]

r2 = 0.5
D2 = 2.0
mean2 = [3, 7, -2, 4, 6] + np.array(range(1, n_feature + 1)) * 0.5
V2 = [[D2 * r2 ** abs(i - j) for j in range(n_feature)] for i in range(n_feature)]

r3 = -0.7
D3 = 1.0
mean3 = [3, 7, -2, 4, 6] + np.array(range(1, n_feature + 1)) * -0.5
V3 = [[D3 * r3 ** abs(i - j) for j in range(n_feature)] for i in range(n_feature)]

# моделирование обучающей выборки
N1, N2, N3 = 1000, 1200, 900
x1 = np.random.multivariate_normal(mean1, V1, N1).T
x2 = np.random.multivariate_normal(mean2, V2, N2).T
x3 = np.random.multivariate_normal(mean3, V3, N3).T

data_x = np.hstack([x1, x2, x3]).T
data_y = np.hstack([np.zeros(N1), np.ones(N2), np.ones(N3) * 2])

x_train, x_test, y_train, y_test = train_test_split(data_x, data_y, random_state=123, test_size=0.3, shuffle=True)

# здесь продолжайте программу
clf = RandomForestClassifier(max_depth=8, n_estimators=10)
clf.fit(x_train, y_train)

predict = clf.predict(x_test)
Q = np.mean(predict == y_test)

```
