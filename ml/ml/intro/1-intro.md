# Постановка задачи ML

$X^l = \{(x_i, y_i)\}_{i=1}^l$ — размеченная обучающая выборка: набор из $l$ пар «объект–ответ», где $x_i$ — описание объекта (вектор признаков), $y_i$ — известный правильный ответ для него. Верхний индекс $l$ у $X$ — это просто размер выборки, не степень.

Например, для задачи предсказания цены квартиры одна пара выглядит так:

$$x_i = [54, 3, 1, 12], \quad y_i = 8\,500\,000$$

где признаки — площадь (54 м²), этаж (3), наличие балкона (1), расстояние до метро (12 мин), а ответ — цена в рублях. Для задачи классификации спама $y_i \in \{0, 1\}$ (не спам / спам), для задачи распознавания цифр $y_i \in \{0, 1, \ldots, 9\}$.

$F = \{f_j(x)\}_{j=1}^n$ — пространство признаков: полный набор из $n$ функций, каждая из которых извлекает одну характеристику объекта. Для той же квартиры:

| $j$ | Признак $f_j$ | Значение $f_j(x_i)$ |
|---|---|---|
| 1 | площадь, м² | 54 |
| 2 | этаж | 3 |
| 3 | наличие балкона | 1 |
| 4 | расстояние до метро, мин | 12 |

Выбор признаков определяет, что модель вообще «видит» в данных — плохие признаки не исправит никакой алгоритм.

$f_j$ — это необязательно просто «взять столбец из таблицы». Это может быть произвольное преобразование объекта:

- $f_j(x) = \text{площадь} / \text{число комнат}$ — производный признак (площадь на комнату)
- $f_j(x) = \log(\text{цена})$ — логарифмирование, чтобы убрать скошенность распределения
- $f_j(x) = \mathbb{1}[\text{этаж} = 1 \text{ или этаж} = \text{последний}]$ — бинаризация условия
- $f_j(x) = \|\text{embedding}(x)\|$ — норма эмбеддинга, полученного нейросетью

Именно поэтому feature engineering — это по сути конструирование функций $f_j$: чем лучше они отражают структуру задачи, тем проще модели найти закономерность.

**Признаковое описание объекта.** Каждый объект $x$ представлен набором из $n$ признаков. Признак $f_j : X \to D_j$ — это функция, которая отображает объект в некоторое измеримое значение из области $D_j$:

- $D_j = \mathbb{R}$ — вещественный признак (рост, температура, цена)
- $D_j = \{0, 1\}$ — бинарный признак (наличие/отсутствие свойства)
- $D_j = \{v_1, v_2, \ldots\}$ — категориальный признак (цвет, тип, метка)
- $D_j = \mathbb{R}^d$ — векторный признак (координата, embedding)

Значение $f_j(x_i)$ — результат применения $j$-го признака к $i$-му объекту. Совокупность всех признаков даёт **вектор признаков**:

$$x_i = \bigl[f_1(x_i),\; f_2(x_i),\; \ldots,\; f_n(x_i)\bigr] \in D_1 \times D_2 \times \cdots \times D_n$$

где строка матрицы объект–признак соответствует одному объекту. Задача проектирования признаков (feature engineering) — выбрать или сконструировать такие $f_j$, которые несут максимум информации о целевой переменной.

$a(x)$ — формируемая модель, решающая функция (decision function): принимает объект $x$ и возвращает предсказание $\hat{y}$

$y_i = y(x_i)$ — истинный ответ на объект $x_i$, известный из обучающей выборки

$a(x) = g(x, \theta)$ — параметрическая модель, где $g$ — фиксированная форма (например, линейная), а $\theta$ — вектор параметров, которые подбираются в процессе обучения:

$$\theta = [\theta_1, \theta_2, \ldots, \theta_n]$$

На примере линейной функции $g(x, k, b) = k \cdot x + b$, где $k$ — наклон (коэффициент при $x$), $b$ — свободный член (сдвиг). Наблюдаемый ответ содержит шум: $y = k \cdot x + b + \varepsilon$, где $\varepsilon \sim \mathcal{N}(0, \sigma^2)$ — гауссовский шум. Параметры модели: $\theta = [k, b]$.

Критерий качества — функция потерь $L$, измеряющая ошибку одного предсказания:

$$L(a, x) = |a(x) - y(x)| \quad \text{— абсолютная ошибка (MAE)}$$
$$L(a, x) = (a(x) - y(x))^2 \quad \text{— квадратичная ошибка (MSE)}$$

Средний эмпирический риск — усреднение потерь по всей выборке, именно его минимизируют при обучении:

$$Q(a, X^l) = \frac{1}{l} \sum_{i=1}^l L(a, x_i)$$

где $l$ — размер выборки, $a$ — модель, $X^l$ — обучающая выборка.

Для квадратичной потери и линейной модели $Q$ раскрывается явно:

$$Q = \frac{1}{l} \sum_{i=1}^l (k \cdot x_i + b - y_i)^2$$

Минимум находится из условия равенства нулю частных производных (метод наименьших квадратов):

$$\frac{\partial Q}{\partial k} = \frac{2}{l} \sum_{i=1}^l (k \cdot x_i + b - y_i) \cdot x_i = 0$$

$$\frac{\partial Q}{\partial b} = \frac{2}{l} \sum_{i=1}^l (k \cdot x_i + b - y_i) = 0$$

где $\partial Q / \partial k$ — скорость изменения ошибки при изменении наклона, $\partial Q / \partial b$ — при изменении сдвига. В точке минимума обе производные равны нулю — это и есть оптимальные параметры $k^*, b^*$.

Общая задача ML — поиск модели, описывающей зависимость в данных. Она сводится к параметрической оптимизации: найти $\theta^* = \arg\min_\theta Q(g(x, \theta), X^l)$.

# задачи

Задачи подразделяются на:

- задачи регрессии — прогноз
- задачи классификации — определение объектов
- задачи ранжирования — обработка запросов

**обучающая выборка** представлена в виде векторов

$x_i = [x_{i1}, x_{i2}, \ldots, x_{in}]$

$X = \begin{bmatrix} x_1 \\ x_2 \\ \vdots \\ x_l \end{bmatrix} = \begin{bmatrix} x_{11} & x_{12} & \cdots & x_{1n} \\ \vdots & \vdots & \ddots & \vdots \\ x_{l1} & x_{l2} & \cdots & x_{ln}\end{bmatrix}$

$n$ — число признаков
$l$ — число объектов
входные данные, целевые значения

$Y = [y_1, y_2, \ldots, y_l]^T$

При классификации

$y \in \{-1, 1\}$ — бинарная классификация

$y \in \{1, 2, 3, \ldots, m\}$ — многоклассовая классификация

$y \in \{0, 1\}^M$ — многометочная классификация (overlapping classes), $M$ — число классов

$y \in \mathbb{R}$ — одномерный случай регрессии

$y \in \mathbb{R}^m$ — многомерный случай регрессии

$X^l = \{(x_i, y_i)\}_{i=1}^l$ — размеченная обучающая выборка

**Признаки на практике — не просто сырые данные.** Входной объект $x$ может быть изображением, текстом, сигналом датчика — и напрямую кормить модель пикселями или байтами неэффективно. В реальных задачах большая часть работы — в том, как превратить сырой $x$ в информативный вектор $[f_1(x), \ldots, f_n(x)]$. Для этого используют три подхода:

- **Feature extraction** — автоматическое извлечение признаков из сырых данных. Например, свёрточная сеть сама учится выделять края, текстуры и формы из пикселей; wav2vec извлекает акустические эмбеддинги из аудио
- **Вторичные признаки** — признаки, построенные поверх первичных вручную. Из пикселей → гистограмма градиентов (HOG); из транзакций → «сумма покупок за последние 7 дней»; из текста → TF-IDF
- **Feature engineering** — общий термин для ручной работы с признаками с опорой на предметную область: конструирование новых $f_j$, отбор информативных, устранение коррелированных и нормализация

После того как признаки выбраны, их числовое представление нужно привести к единому масштабу — это **feature scaling**. Признаки «возраст» $\in [0, 100]$ и «зарплата» $\in [0, 1\,000\,000]$ несопоставимы по величине: модели, опирающиеся на расстояния или градиенты, будут игнорировать «возраст» просто потому что его числа меньше. Два стандартных способа:

$$\tilde{f}_j = \frac{f_j - \mu_j}{\sigma_j} \quad \text{(стандартизация, z-score — нулевое среднее, единичная дисперсия)}$$

где $\mu_j = \frac{1}{l}\sum_{i=1}^l f_j(x_i)$ — среднее признака по выборке, $\sigma_j$ — его стандартное отклонение, $\tilde{f}_j$ — нормализованное значение признака.

$$\tilde{f}_j = \frac{f_j - \min_j}{\max_j - \min_j} \quad \text{(нормализация min-max — сжатие в } [0, 1]\text{)}$$

где $\min_j = \min_i f_j(x_i)$, $\max_j = \max_i f_j(x_i)$ — минимум и максимум признака по выборке.

Scaling — часть предобработки, а не конструирования признаков: он не меняет того, _что_ измеряется, только _в каких единицах_.
