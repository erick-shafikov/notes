# SVRG — Stochastic Variance Reduced Gradient

SGD обновляет веса по градиенту одного случайного объекта — это быстро, но шумно: дисперсия градиента не убывает к нулю при приближении к оптимуму, поэтому приходится уменьшать шаг обучения, что замедляет сходимость. SVRG (Johnson & Zhang, 2013) решает эту проблему, не увеличивая размер батча.

## Контекст: сравнение методов

Рассматриваем задачу минимизации суммы:

$$f(x) = \frac{1}{n} \sum_{i=1}^{n} f_i(x)$$

где $x \in \mathbb{R}^d$ — вектор параметров модели, $f_i(x)$ — функция потерь на $i$-м объекте обучающей выборки, $n$ — размер выборки. Запрос $\nabla f_i(x)$ стоит $O(1)$, запрос полного градиента $\nabla f(x) = \frac{1}{n}\sum_i \nabla f_i(x)$ — $O(n)$, так как требует прохода по всем объектам.

Предположения:

- каждая $f_i$ является [$L$**-гладкой**](../../../math/math-an/l-smoothness.md): $\|\nabla f_i(x) - \nabla f_i(y)\| \leq L\|x - y\|$ для любых $x, y$ — градиент не меняется слишком резко
- общая $f$ является $\mu$**-сильно выпуклой**: $f(y) \geq f(x) + \langle \nabla f(x), y - x\rangle + \frac{\mu}{2}\|y - x\|^2$ — функция не просто выпукла, но и обязана расти хотя бы как парабола с кривизной $\mu$ в любом направлении. При большом $\mu$ функция похожа на крутую узкую чашу — оптимум хорошо локализован, сходимость быстрая. При малом $\mu \to 0$ — широкое плоское блюдце, градиент слабый, сходимость медленная. Функция без регуляризации может быть выпуклой, но не сильно выпуклой; добавление L2-члена $\frac{\lambda}{2}\|w\|^2$ физически делает её $\lambda$-сильно выпуклой, прибавляя параболу с кривизной $\lambda$.

**Число обусловленности** $\kappa = L / \mu$ — безразмерная характеристика сложности задачи: $L$ ограничивает кривизну сверху (насколько быстро может меняться градиент), $\mu$ — снизу (насколько функция «изогнута» в самом пологом направлении). Большое $\kappa$ означает, что функция в одних направлениях крутая, в других почти плоская — классический пример «вытянутый эллипсоид», где градиентный спуск движется зигзагами и сходится медленно.

| Метод    | Правило обновления                                                   | Сложность                         |
| -------- | -------------------------------------------------------------------- | --------------------------------- |
| GD       | $x_{k+1} = x_k - \eta \cdot \frac{1}{n}\sum_{i=1}^n \nabla f_i(x_k)$ | $n\kappa \log(1/\varepsilon)$     |
| SGD      | $x_{k+1} = x_k - \eta \nabla f_{i_k}(x_k)$                           | $1/(\mu\varepsilon)$              |
| **SVRG** | см. ниже                                                             | $(n + \kappa)\log(1/\varepsilon)$ |

где $\eta > 0$ — шаг обучения (learning rate), $i_k \sim \mathrm{Uniform}[n]$ — случайно выбранный индекс объекта на шаге $k$, $\varepsilon$ — желаемая точность по значению функции.

GD сходится линейно, но каждый шаг стоит $O(n)$ — при большой выборке слишком дорого. SGD делает дешёвые шаги, но из-за постоянного шума в градиенте сходится лишь как $O(1/(\mu\varepsilon))$ — медленно при малых $\varepsilon$. SVRG достигает $(n + \kappa)\log(1/\varepsilon)$: вместо произведения $n\kappa$ стоит **сумма**, что при больших $n$ и $\kappa$ значительно лучше.

## Идея: редуцирование дисперсии

Корень проблемы SGD — постоянная дисперсия стохастического градиента: даже вблизи оптимума $\nabla f_i(x)$ шумит с той же амплитудой, что и в начале обучения. Чтобы шум убывал автоматически, вводят **скорректированную оценку градиента**:

$$v_i = \nabla f_i(x) - \nabla f_i(\tilde{x}) + \nabla f(\tilde{x})$$

где $x$ — текущая точка параметров, $\tilde{x}$ — **опорная точка** (snapshot), зафиксированная в начале текущей эпохи и обновляемая редко, $\nabla f(\tilde{x}) = \frac{1}{n}\sum_{j=1}^n \nabla f_j(\tilde{x})$ — точный полный градиент в опорной точке, вычисленный один раз за эпоху.

Оценка $v_i$ обладает двумя ключевыми свойствами:

**Несмещённость.** Взяв математическое ожидание по $i \sim \mathrm{Uniform}[n]$, первые два члена дают одно и то же в среднем и сокращаются:

$$\mathbb{E}_i[v_i] = \mathbb{E}_i[\nabla f_i(x)] - \mathbb{E}_i[\nabla f_i(\tilde{x})] + \nabla f(\tilde{x}) = \nabla f(x) - \nabla f(\tilde{x}) + \nabla f(\tilde{x}) = \nabla f(x)$$

Оценка указывает в правильном направлении в среднем — точно так же, как и полный градиент.

**Малая дисперсия вблизи оптимума.** Когда $x$ и $\tilde{x}$ оба близки к $x^*$, из $L$-гладкости $f_i$:

$$\|\nabla f_i(x) - \nabla f_i(\tilde{x})\| \leq L\|x - \tilde{x}\| \approx 0$$

Разность $\nabla f_i(x) - \nabla f_i(\tilde{x})$ стремится к нулю, и оценка $v_i \approx \nabla f(\tilde{x}) \approx \nabla f(x)$ становится почти детерминированной. Именно поэтому шаг обучения $\eta$ можно держать **фиксированным** — не нужно затухающего расписания как в SGD.

## Алгоритм

SVRG работает в двух вложенных циклах:

```
инициализировать x̃₀ (начальное приближение), m (длина внутреннего цикла)
для s = 0, 1, 2, ... (внешний цикл — эпохи):
    вычислить ∇f(x̃ₛ) = (1/n) Σᵢ ∇fᵢ(x̃ₛ)   ← O(n): полный проход по выборке
    установить x₀ = x̃ₛ                        ← начать внутренний цикл из снапшота
    для k = 0, 1, ..., m-1 (внутренний цикл):
        выбрать Iₖ ~ Uniform[n]                ← случайный объект
        vₖ = ∇f_Iₖ(xₖ) - ∇f_Iₖ(x̃ₛ) + ∇f(x̃ₛ) ← O(1): два градиента + сохранённый
        xₖ₊₁ = xₖ - η·vₖ                      ← шаг с редуцированным градиентом
    x̃ₛ₊₁ = xₙ  (случайный снапшот из {x₀,...,xₘ₋₁}) ← обновить опорную точку
```

| Переменная              | Роль                                                                                                                            |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| $\tilde{x}_s$           | опорная точка (snapshot) — фиксируется в начале каждой эпохи $s$, обновляется раз в $m$ шагов                                   |
| $x_k$                   | текущая точка параметров внутри эпохи, меняется на каждом шаге внутреннего цикла                                                |
| $s$                     | номер эпохи (внешний цикл), счётчик полных проходов по логике алгоритма                                                         |
| $k$                     | шаг внутри эпохи, $k = 0, \ldots, m-1$                                                                                          |
| $m$                     | длина внутреннего цикла — гиперпараметр; теория рекомендует $m \geq 16\kappa$                                                   |
| $n$                     | размер обучающей выборки                                                                                                        |
| $\nabla f(\tilde{x}_s)$ | точный полный градиент в опорной точке, вычисляется один раз за эпоху ценой $O(n)$ и хранится до следующего обновления снапшота |
| $I_k$                   | случайный индекс объекта на шаге $k$, сэмплируется равномерно из $\{1, \ldots, n\}$                                             |
| $v_k$                   | редуцированная оценка градиента: несмещённая, но с малой дисперсией вблизи оптимума                                             |
| $\eta$                  | шаг обучения (learning rate) — фиксированный, не убывает в отличие от SGD                                                       |

**Внешний цикл** (индекс $s$): каждая итерация начинается с вычисления точного полного градиента $\nabla f(\tilde{x}_s)$ в текущей опорной точке $\tilde{x}_s$. Это стоит $O(n)$, но делается лишь раз за эпоху.

**Внутренний цикл** (индекс $k$): $m$ шагов по одному случайному объекту. На каждом шаге считаются два стохастических градиента — $\nabla f_{I_k}(x_k)$ в текущей точке и $\nabla f_{I_k}(\tilde{x}_s)$ в опорной — и добавляется сохранённый $\nabla f(\tilde{x}_s)$. Стоимость шага $O(1)$.

Стоимость одной внешней итерации: $O(n) + O(m) = O(n + m)$. При $m \sim \kappa$ суммарная стоимость для достижения точности $\varepsilon$ составляет $O((n + \kappa)\log(1/\varepsilon))$.

## Гарантия сходимости

**Теорема (Johnson & Zhang, 2013).** При $\eta = \dfrac{1}{8L}$ и $m \geq 16\kappa$ выполняется:

$$\mathbb{E}[f(\tilde{x}_{s+1})] - f^* + \mu\,\mathbb{E}\|\tilde{x}_{s+1} - x^*\|^2 \leq \frac{1}{2}\Bigl(\mathbb{E}[f(\tilde{x}_s)] - f^* + \mu\|\tilde{x}_s - x^*\|^2\Bigr)$$

где $f^* = \min_x f(x)$ — оптимальное значение функции, $x^*$ — точка минимума, $\mathbb{E}[\cdot]$ — математическое ожидание по случайным выборкам индексов.

Левая часть — **мера субоптимальности**: сумма зазора по значению $f(\tilde{x}_{s+1}) - f^*$ и взвешенного квадрата расстояния до оптимума $\mu\|\tilde{x}_{s+1} - x^*\|^2$. Неравенство говорит, что эта мера уменьшается в 2 раза на каждой внешней итерации, независимо от номера итерации. Значит, для достижения $\varepsilon$-точности нужно $O(\log(1/\varepsilon))$ внешних итераций.

## Фундаментальная лемма SVRG

В основе доказательства лежит неравенство, связывающее дисперсию стохастического градиента с зазором субоптимальности. Из $L$-гладкости $f_i$ и выпуклости следует:

$$\mathbb{E}_i\|\nabla f_i(x) - \nabla f_i(x^*)\|^2 \leq 2L\bigl(f(x) - f(x^*)\bigr)$$

где математическое ожидание берётся по $i \sim \mathrm{Uniform}[n]$. Смысл: разброс стохастических градиентов $\|\nabla f_i(x) - \nabla f_i(x^*)\|^2$ ограничен сверху зазором субоптимальности $f(x) - f^*$, умноженным на $2L$. Чем ближе $x$ к оптимуму, тем меньше дисперсия — и тем точнее становится оценка $v_i$.

## Сравнение с SAG

SAG (Stochastic Average Gradient) тоже редуцирует дисперсию, но другим способом — хранит последний вычисленный градиент для каждого из $n$ объектов и усредняет их. SVRG хранит только одну опорную точку $\tilde{x}_s$ и полный градиент в ней:

|                | SAG                                             | SVRG                                                    |
| -------------- | ----------------------------------------------- | ------------------------------------------------------- |
| Память         | $O(nd)$ — хранит $n$ градиентов размерности $d$ | $O(d)$ — только $\tilde{x}_s$ и $\nabla f(\tilde{x}_s)$ |
| Стоимость шага | $O(d)$                                          | $O(d)$ (+ $O(nd)$ раз в $m$ шагов)                      |
| Сходимость     | линейная                                        | линейная                                                |

SVRG предпочтительнее при большом $n$, когда хранить $n$ градиентов накладно. SAG удобнее, когда полный проход по выборке дорог и хочется обновлять опорную информацию непрерывно.
