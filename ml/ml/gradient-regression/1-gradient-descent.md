# Градиентны спуск

**Метод Робинса-Монро** (1951) — общий теоретический фреймворк для стохастической аппроксимации: итеративного нахождения корня уравнения $\mathbb{E}[f(x, \xi)] = 0$ или минимума функции, когда функция недоступна напрямую, а наблюдается только зашумлённо. Градиентный спуск и SGD — частные случаи этого фреймворка. Ключевой результат теории: алгоритм вида $x_{t+1} = x_t - \eta_t \cdot g_t$ сходится к оптимуму почти наверное, если шаги $\eta_t$ удовлетворяют двум условиям:

$$\sum_{t=1}^{\infty} \eta_t = \infty, \qquad \sum_{t=1}^{\infty} \eta_t^2 < \infty$$

Первое условие гарантирует, что суммарный пройденный путь бесконечен — алгоритм способен добраться до любой точки пространства и не застрянет далеко от оптимума. Второе ограничивает накопленный шум: сумма квадратов шагов конечна, поэтому дисперсия оценки постепенно убывает и алгоритм стабилизируется. Пример такой последовательности: $\eta_t = 1/t$ — условия выполнены, так как $\sum 1/t = \infty$ (гармонический ряд расходится) и $\sum 1/t^2 < \infty$ (ряд Базеля сходится). На практике константный шаг ($\eta_t = \text{const}$) нарушает второе условие, но часто работает лучше из-за более быстрой сходимости на ранних итерациях.

Так как признаков может быть много и матрица $(X^T X)$ может быть вырожденной или плохо обусловленной, а также её обращение вычислительно затратно, то вместо аналитического решения $w = (X^T X)^{-1} X^T y$ используют градиентные алгоритмы. Нахождение минимума — производная (в одномерном случае). В многомерном случае используется градиент — вектор частных производных. Градиент показывает направление наискорейшего роста функции, поэтому для поиска минимума движение происходит в сторону антиградиента. Градиент — это значение производной (частных производных) в точке. Формула обновления: $x_{n+1} = x_n - λ * grad f(x_n)$, где $λ$ — шаг обучения. Смысл в том, что вместо решения уравнения $f'(x)=0$ (что может быть сложно или невозможно аналитически) используется итеративное приближение к минимуму.

```python
import numpy as np

def f(x):
    return x*x - 5*x + 5

def df(x):
    return 2*x - 5

N = 20     # число итераций
xx = 0      # начальное значение
lmd = 0.9  # шаг сходимости

for i in range(N):
    xx = xx - lmd*df(xx)    # изменение аргумента на текущей итерации
```

```python
import numpy as np


# исходная функция, которую нужно аппроксимировать моделью a(x)
def func(x):
    return 0.1 * x ** 2 - np.sin(x) + 5.


coord_x = np.arange(-5.0, 5.0, 0.1)  # значения по оси абсцисс [-5; 5] с шагом 0.1
coord_y = func(coord_x)  # значения функции по оси ординат

eta = np.array([0.1, 0.01, 0.001, 0.0001])  # шаг обучения для каждого параметра w0, w1, w2, w3
w = np.array([0., 0., 0., 0.])  # начальные значения параметров модели
N = 200  # число итераций градиентного алгоритма

X = np.power(coord_x.reshape(-1, 1), range(4)) # превращает одномерный x в матрицу полиномиальных признаков степени 3 (включая константу)

for _ in range(N):
    y_pred = X @ w
    w -= eta * 2 * ((y_pred - coord_y) * X.T).mean(1)

Q = np.square(y_pred - coord_y).mean()
```

При расчете градиента также используют нормирование (на практике применяется при обучении нейросетей — RNN, LSTM, Transformer — для защиты от взрывного градиента; в обычном градиентном спуске не нужно): $grad f(x, y, z, ...)={∂f/∂x, ∂f/∂y, ∂f/∂z, ...}$ — градиент функции. $|grad f(x,y,z,...)| = \sqrt{(∂f/∂x)^2 + (∂f/∂y)^2 + (∂f/∂z)^2 + ...}$ — его норма. Тогда нормированный градиент: $grad f / |grad f|$, и обновление может быть записано как $x_{n+1} = x_n - λ * (grad f(x_n) / |grad f(x_n)|)$. Таким образом движение всегда происходит в сторону антиградиента (уменьшения функции). Шаг λ может быть константным или изменяться по итерациям, например: $λ_n = 1 / min(n+1, m)$, где m — ограничение на минимальный шаг.
