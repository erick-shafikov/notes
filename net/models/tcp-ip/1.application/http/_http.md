# http протокол

hypertext protocol - работает в режиме запрос-ответ, использует протокол tcp на транспортном, ip на интернет - уровне . Порты по умолчанию http - 80, httpS - 433

- Обмен идет в формате запрос-ответ
- нет состояния (не знает о предыдущих запросах)
- TCP-сессия в ходе которой один раз устанавливается соединение
- Три агента:
- - клиент
- - сервер (apache/nginx),
- - прокси

По нему передается не только страницы, но и файлы

# http сессия

Фазы:

- Клиент устанавливает TCP соединения
- - на сервере порт 80 (8000, 8080), url страницы содержит доменное имя и порт
- Клиент отправляет запрос и ждёт ответа
- Сервер обрабатывает запрос и посылает ответ

TCP не закрывается

# https

работает на основе tls (то в свою очередь tcp). Основной порт 443, ссылка с префиксом https:// Установка соединение:

- tcp (3 шага)
- tls (client hello) - по нему и передаются сообщения
