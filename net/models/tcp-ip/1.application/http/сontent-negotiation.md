# Согласование контента

Механизм при котором по одному и тому же url будет представлен разный контент. Механизм реализуется с помощью:

- согласовании на стороне сервера - с помощью заголовков
- согласование на стороне клиента - с помощью кодов ответа 300, 406

# согласование на стороне сервера

С клиента идет запрос с заголовками Accept, Accept-Encoding,Accept-Language, также могут быть включена client hints заголовки. Заголовок Vary может быть предоставлен для кеширования ответа. Сервер отвечает относительно этих заголовков.

# согласование на стороне клиента

Происходит в несколько стадий:

- клиент отправляет запрос с Accept-заголовками
- сервер отвечает с 300 кодом со списком url
- клиент выбирает нужный
