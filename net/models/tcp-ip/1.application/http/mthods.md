# Методы

Каждый метод содержит:

- Заголовки
- Тело сообщения

Методы:

- CONNECT - метод запроса прокси сервера если используется ssl для нахождения tcp туннеля

```bash
    CONNECT server.example.com:80 HTTP/1.1
    Host: server.example.com:80
    Proxy-Authorization: basic aGVsbG86d29ybGQ=
```

- OPTIONS – проверяет работоспособность сайта, в ответ придет должен прийти список с доступными методами

```bash
# запрос
curl -X OPTIONS http://example.org -i
# ответ
HTTP/1.1 200 OK
Allow: OPTIONS, GET, HEAD, POST
Cache-Control: max-age=604800
Date: Thu, 13 Oct 2016 11:45:00 GMT
Expires: Thu, 20 Oct 2016 11:45:00 GMT
Server: EOS (lax004/2813)
x-ec-custom-error: 1
Content-Length: 0
```

- GET – в нем можно передавать параметры через знак ?
- HEAD – проверит заголовки, которые вернуться для get
- POST – имеет тело, при успехе 200 – удачно выполнен, 201 – создан ресурс
- PUT – для загрузки содержимого (не кешируется на сервере) в отличие от POST является идемпотентным, то есть запрос с теме же данными не будет привозить к эффектам
- PATCH – к фрагменту
- DELETE - для удаления ресурса. Статус ответа:
- - 202 - удаление успешно но пока не выполнено
- - 204 - удаление без тела
- - 200 удаление + тело состояния
- TRACE - для отладки соединения
