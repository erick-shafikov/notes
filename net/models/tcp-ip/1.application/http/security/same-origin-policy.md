# Same-origin policy

Политика одного источника определяет как страница может взаимодействовать с ресурсами из другого источника

Ограничения касаются разных протоколов, портов и хостов. Так же блокируется переход

```js
document.domain = "company.com";
```

если используется не супер-домен

Политика одного источника контролирует взаимодействие между двумя разными источниками, например, при использовании XMLHttpRequest или элемента <img>. Эти взаимодействия обычно делятся на три категории:

- Запись данных из других источников обычно разрешена.
- Встраивание данных из разных источников обычно допускается. Примеры приведены ниже.
- Как правило, чтение из других источников запрещено, но доступ к чтению часто становится недоступен при встраивании данных.

Вот несколько примеров ресурсов, которые могут быть встроены из других источников:

- JavaScript с <script src="..."></script>. Сообщения об ошибках синтаксиса доступны только для скриптов, находящихся в том же источнике.
- CSS с использованием <link rel="stylesheet" href="...">. Из-за ослабленных синтаксических правил CSS, для кросс-доменного CSS требуется корректный заголовок Content-Type. Ограничения различаются в зависимости от браузера: IE, Firefox, Chrome, Safari (прокрутите вниз до CVE-2010-0051) и Opera.
- Изображения с тегом <img>. Поддерживаемые форматы изображений: PNG, JPEG, GIF, BMP, SVG.
- Медиафайлы, содержащие <видео> и <аудио>.
- Плагины с <object>, <embed> и <applet>.
- Шрифты с атрибутом @font-face.
- Любой контент, содержащий <frame> и <iframe>. Сайт может использовать заголовок X-Frame-Options, чтобы предотвратить подобное взаимодействие между источниками.

Доступны методы и атрибуты windows:

- window.blur
- window.close
- window.focus
- window.postMessage
- window.closed Read only.
- window.frames Read only.
- window.length Read only.
- window.location Read/write.
- window.opener Read only.
- window.parent Read only.
- window.self Read only.
- window.top Read only.
- window.window
